1. High-Level Overview

This project is a full-stack AI-powered security tool that detects vulnerabilities in Python source code using a hybrid approach: rule-based pattern detectors combined with a Graph Neural Network (GNN) machine learning model. It features a React frontend, FastAPI backend, real-time WebSocket analysis, and an AI chatbot powered by Google Gemini 2.0 for vulnerability explanations.

2. Step-by-Step Process Walkthrough

Each step below describes a stage in the vulnerability detection pipeline, from user input to final report.

1
User Input

The user interacts with the React frontend by either:

  • Pasting Python code into the code editor (CodeEditor.jsx)
  • Uploading .py files via drag-and-drop (FileUpload.jsx)

The frontend sends the code to the backend via POST /api/v1/analyze (REST) or through a WebSocket connection for real-time analysis.

2
Security Middleware

Before any analysis begins, the request passes through three security layers:

LayerFilePurpose
API Key Authauth.pyValidates X-API-Key header
Rate Limiterrate_limiter.pyPrevents abuse via SlowAPI
Input Validatorsvalidators.pySanitizes filenames, checks file size, validates extensions
3
AST Parsing

The ASTParser (ast_parser.py) uses Python's built-in ast module to parse source code into an Abstract Syntax Tree. Each node — function call, assignment, import, etc. — becomes a structured node for downstream analysis.

4
Graph Building

The GraphBuilder (graph_builder.py) transforms the AST into a Code Property Graph (CPG) containing:

  • Control flow edges — how execution flows between statements
  • Data flow edges — how variables pass data between nodes
  • Call edges — function call relationships
  • Node features — 128-dimensional embeddings via EmbeddingGenerator
5
Taint Analysis

The TaintAnalyzer (taint_analyzer.py) tracks how untrusted data (user input, network data, file reads) flows through the code to dangerous sinks (eval, exec, SQL queries, system commands). It identifies complete source → sink paths that represent exploitable vulnerabilities.

6
Sanitizer Detection

The SanitizerDetector (sanitizer_detector.py) identifies security mitigations already present in the code:

  • Input validation functions
  • Parameterized queries
  • HTML escaping / encoding
  • Type checks and whitelists

Found sanitizers reduce vulnerability severity in the next step.

7
Rule-Based Detection

Nine specialized detectors scan the AST nodes for known vulnerability patterns:

DetectorVulnerabilitySeverity
eval_exec.pyeval()/exec() Injection🔴 Critical
command_injection.pyos.system(), subprocess with shell=True🔴 Critical
deserialization.pypickle.loads(), unsafe YAML🔴 Critical
hardcoded_secrets.pyAPI keys, passwords in code🟠 High
cryptography.pyWeak ciphers, MD5/SHA1🟡 Medium
xxe.pyXML External Entity attacks🟠 High
redos.pyRegex Denial of Service🟡 Medium
xss_detector.pyCross-Site Scripting🟠 High
logic_flaws.pyAuthentication / Logic bugs🟠 High
8
Severity Adjustment

The SeverityAdjuster (severity_adjuster.py) cross-references rule detections against sanitizers and taint paths:

  • If a sanitizer fully mitigates a vulnerability → marked SAFE (skipped)
  • If a taint path exists but is partially sanitized → severity reduced
  • Vulnerabilities without taint paths may have severity lowered
9
GNN Model Inference

The VulnerabilityGNN (gnn_model.py) processes the code graph through:

  • Input Projection — 128 → 256 dimensional linear layer
  • Multi-Head Graph Attention — 3 layers × 4 heads, learns which nodes are important
  • Global Attention Pooling — aggregates all node features into a single graph embedding
  • Classification Head — outputs probabilities for 7 vulnerability classes

The pretrained model weights are stored in vulnerability_gnn.pt (~10 MB).

10
Result Combination

The VulnerabilityInference engine (inference.py) combines both detection methods using a weighted formula:

Final Confidence = (Rule-Based × 0.7) + (ML Model × 0.3)
  • Vulnerabilities above the confidence threshold are reported
  • ML-only detections require >80% confidence
  • Results are sorted by confidence (highest first)
11
Explainability & Visualization

The attention weights from the GNN model are used to highlight suspicious code lines:

  • AttentionMapper — maps node attention weights to code regions
  • Highlighter — generates highlighted line data with importance scores

The frontend renders these via the ResultsPanel and TraceVisualizer components, showing exactly which lines are most suspicious.

12
AI Chatbot Explanation

Users can ask the AI Assistant (FloatingChatBot.jsx) to explain any detected vulnerability. The backend chat.py forwards the conversation to the OpenRouter API (Google Gemini 2.0 Flash) with full vulnerability context for multi-turn discussions.

This provides natural-language explanations of why code is dangerous, how it can be exploited, and how to fix it.
